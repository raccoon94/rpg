## 핵심 설계 3줄

1. **유대(호감도)가 곧 세계를 구하는 힘** — 미연시 시스템이 메인 스토리의 해답 (메커니즘=세계관).
2. **환령 = 죽은 영웅의 영혼, 노바 = 유대를 잃은 선대 소환사** — 1차 반전(5장), 대반전(8장), 역전(9장: 캐릭터들이 주인공을 구한다).
3. **확장성**: 메인 고정 캐릭터는 린·카이론 2인뿐. 나머지는 주연 슬롯, 신규 캐릭터는 온보딩 템플릿 8항목만 채우면 합류 (5/9/10장 전원 연출은 보유 캐릭터 순회 — 데이터 주도).

## 구현 메모 (코드 반영 시)

- VN 대본은 기존 `playVN([{who,text}...])` 포맷에 그대로 들어감. 추가 필요: `face`(표정), `choices`(선택지), `G.flags`(선택 기억).
- 에피소드는 `CHARACTERS[이름].episodes`에 `{title, need, mainReq, script, reward}` 형태로.
- 메인 스토리는 `MAIN_STORY` 배열 신설: `{ch, title, stage, focus, script, reward}`.
- 9장 "손 내밀기"·5장 "전원 반응"은 `CHARACTERS`에 `ch5Line`/`ch9Line` 필드 추가 후 보유 캐릭터 순회 — 신규 캐릭터 자동 합류.
- 수익화: 고백/엔딩은 절대 과금 잠금 금지. 파는 것은 코스튬·호감도 가속·픽업만 (00 문서 금지 목록).
